what maps well

the strong part of the idea.

motor command history gives pitch contour, rough loudness, rapid spool changes, and imbalance between motors. that is enough to fake convincing whoop behaviour, especially once you add a turbulence layer and a bit of frame buzz.

what stays approximate

where the model stops being truth.

the log does not contain actual mic position, room reflections, camera mount resonance, prop geometry, or true acoustic wind. if rpm telemetry is missing, pitch is estimated from motor command rather than measured rotor speed.

deployment path

keep it simple first.

github pages is enough for this current browser-only build. a cloudflare worker only becomes useful if you want raw log decoding helpers, muxed video export jobs, saved presets, or a shareable render api.
